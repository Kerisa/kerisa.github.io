CrackMe 反汇编练习之 092

作为练手的 160 个 CrackMe 系列整理分析

CrackMe 来源：【反汇编练习】160个CrackME索引目录1~160建议收藏备用

这个 CrackMe 使用的是 Blowfish 对称加密算法，算法本身并不复杂，只要将几个关键函数逆向后将程序的流程搞清即可。有个 upx 壳，很好脱就不说了，放着不管也无所谓。

00401550

这是按钮的事件处理函数，可通过字符串快递定位。

里面取出 code 并 wprintf 成两个八位十六进制数，传给 004010D0 进行解密，随后将结果与保存在 0x4099F0 和 0x4099EC 处的值进行比较，两者都相等则成功。

004010D0

Blowfish 解密函数。传入了密码盒和 A、B 两个双字作为参数，代码不长。C 代码列在最后。

解密函数搞定后应该要找到加密的函数，在 0x4099F0 和 0x4099EC 下断可以发现，这两个值并不是写死而是程序启动时生成的，同样后面也能发现密码盒也是启动时初始化的。

00401130

通过在密码盒地址 0x408980 下断可以跟踪到，这个是 Blowfish 的密钥初始化函数。传入了密码盒，一个字符串及其长度作为参数。Blowfish 的密钥表含有 P-Box 和 S-Box 两部分。结构如下：

代码大致逻辑如下：

  1. 复制原始 S-Box
  2. 取原始 P-Box，将其与传入的密钥字符串运算后再复制
  3. 以 0 为初值，使用当前的密码盒循环加密 P-Box，每次的结果都会赋值给 P-Box 相应位置覆盖原来的值
  4. 继承上一步的最后两个加密结果，用相同方法循环加密 S-Box

00401070

在上面的初始化函数 00401130 中会调用到 00401070，即 Blowfish 的加密函数，代码与解密函数 001040D0 相似，这里就不贴了，后面有 C 版本。

00401230

这里包含了程序的初始化流程，主要有以下步骤：

  1. 从注册表 Software\Microsoft\Windows\CurrentVersion 下的 ProductId 取出产品 ID，没有则用 “94940361391” 代替
  2. 用产品 ID 初始化 Blowfish 密码盒
  3. 传入 0x68736966, 0x776f6c62 进行加密，其结果保存到 0x4099F0 和 0x4099EC，wprintf 后就是界面上的 Computer ID
  4. 用固定字符串 “ChinaCrackingGroup” 继续初始化 Blowfish 密码盒
  5. （在对话框初始化时）从 0x4099F0 和 0x4099EC 中取出数据进行加密，得到的结果写回，这就是最开始与注册码进行比较的值
  6. （在对话框初始化时）用固定字符串 “CrackingForFun” 继续初始化 Blowfish 密码盒

因为程序会将我们填入的注册码进行解密后与其比较，那么在写注册机时只要将上面第 5 步中最后的值再做一次加密就好了。
